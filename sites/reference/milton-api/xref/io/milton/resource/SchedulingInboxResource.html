22  /**
23   * 4.2. Scheduling Inbox Collection
24  
25  
26     A scheduling Inbox collection contains copies of incoming scheduling
27     messages.  These may be requests sent by an Organizer, or replies
28     sent by an Attendee in response to a request.
29  
30     A scheduling Inbox collection MUST report the DAV:collection and
31     CALDAV:schedule-inbox XML elements in the value of the DAV:
32     resourcetype property.  The element type declaration for CALDAV:
33     schedule-inbox is:
34  
35        <!ELEMENT schedule-inbox EMPTY>
36  
37     Example:
38  
39        <D:resourcetype xmlns:D="DAV:">
40          <D:collection/>
41          <C:schedule-inbox xmlns:C="urn:ietf:params:xml:ns:caldav"/>
42        </D:resourcetype>
43  
44     Scheduling Inbox collections MUST only contain calendar object
45     resources that obey the restrictions specified in iTIP [RFC5546].
46     Consequently, scheduling Inbox collections MUST NOT contain any types
47     of collection resources.  Restrictions defined in Section 4.1 of
48  
49  
50  
51  Daboo & Desruisseaux     Expires April 28, 2011                [Page 13]
52  
53  
54  Internet-Draft        CalDAV Scheduling Extensions          October 2010
55  
56  
57     CalDAV "calendar-access" [RFC4791] on calendar object resources
58     contained in calendar collections (e.g., "UID" uniqueness) don't
59     apply to calendar object resources contained in a scheduling Inbox
60     collection.  Thus, multiple calendar object resources contained in a
61     scheduling Inbox collection can have the same "UID" property value
62     (i.e., multiple scheduling messages for the same calendar component).
63  
64     New WebDAV ACL [RFC3744] privileges can be set on the scheduling
65     Inbox collection to control from whom the Calendar User associated
66     with the scheduling Inbox collection will accept scheduling messages
67     from.  See Section 13.1 for more details.
68  
69     A scheduling Inbox collection MUST NOT be a child (at any depth) of a
70     calendar collection resource.
71  
72     The following WebDAV properties specified in CalDAV "calendar-access"
73     [RFC4791] MAY also be defined on scheduling Inbox collections:
74  
75        CALDAV:calendar-timezone - when present this contains a time zone
76        that the server can use when calendar date-time operations are
77        carried out, for example when a time-range CALDAV:calendar-query
78        REPORT is targeted at a scheduling Inbox collection.
79  
80        CALDAV:supported-calendar-component-set - when present this
81        indicates the allowed calendar component types for scheduling
82        messages delivered to the scheduling Inbox collection.
83  
84        CALDAV:supported-calendar-data - when present this indicates the
85        allowed media types for scheduling messages delivered to the
86        scheduling Inbox collection.
87  
88        CALDAV:max-resource-size - when present this indicates the maximum
89        size of a resource in octets that the server is willing to accept
90        for scheduling messages delivered to the scheduling Inbox
91        collection.
92  
93        CALDAV:min-date-time - when present this indicates the earliest
94        date and time (in UTC) that the server is willing to accept for
95        any DATE or DATE-TIME value in scheduling messages delivered to
96        the scheduling Inbox collection.
97  
98        CALDAV:max-date-time - when present this indicates the latest date
99        and time (in UTC) that the server is willing to accept for any
100       DATE or DATE-TIME value in scheduling messages delivered to the
101       scheduling Inbox collection.
102 
103       CALDAV:max-instances - when present this indicates the maximum
104       number of recurrence instances in scheduling messages delivered to
105 
106 
107 
108 Daboo & Desruisseaux     Expires April 28, 2011                [Page 14]
109 
110 
111 Internet-Draft        CalDAV Scheduling Extensions          October 2010
112 
113 
114       the scheduling Inbox collection.
115 
116       CALDAV:max-attendees-per-instance - when present this indicates
117       the maximum number of ATTENDEE properties in any instance of
118       scheduling messages delivered to the scheduling Inbox collection.
119
